11 //! @section Overview
12 //!
13 //! This script (csv-to-json.awk) interprets a basic 4-column CSV into a JSON entities file. This is "the old method", but facilitates hand-tweaking if required. In essence, the four columns of the file are:
14 //!
15 //! 1. host/array name
16 //! 2. device type: "host" or "array" (without quotes)
17 //! 3. WWPN
18 //! 4. optional unique name of the WWPN within the (column 1) host
19 //!
20 //! For example:
21 //!
22 //! | name | type | WWPN | (optional) name |
23 //! | ------------- | ----- | :---------------------: | :-------------- |
24 //! | MPV123456-001 | host | 2001001560123456 | MPV123456-001 |
25 //! | MPV123456-013 | host | 2013001560123456 | MPV123456-013 |
26 //! | NetApp-123456 | array | 500a098598123456 | |
27 //! | NetApp-123456 | array | 500a098698123456 | |
28 //! | Oracle01 | host | 10:00:00:00:c9:12:34:56 | Oracle01-A |
29 //! | Oracle01 | host | 10:00:00:00:c9:12:34:57 | Oracle01-B |
30 //!
31 //! @section running Running this Script
32 //!
33 //! This AWK script should work with gawk.exe from the UnxUtils project on
34 //! sourceforge.net (which offers windows binaries without need of cygwin) and
35 //! is regression-tested using basic awk on BSD-based operating systems.
36 //!
37 //! gawk.exe -f csv-to-json.awk OrderedTuples.csv > customer-name.json
38 //!
39 //! @section cmdline Commandline Variables:
40 //!
41 //! - -v SKIPARRAY={something} will avoid exporting the array which contains iomodules
42 //! - -v SKIPHOST={something} will avoid exporting the host which contains hbas
43 //!
44 //! @section Example Usage (2 switches)
45 //!
46 //! plink.exe -l username -pw pAssw0rd 192.168.1.44 'show device-alias database' > SW44.dad
47 //! plink.exe -l username -pw pAssw0rd 192.168.1.42 'zoneshow' > SW42.zoneshow
48 //! java.exe -jar vw4tool.jar -N SW44.dad -N SW42.zoneshow -oOrderedTuples.csv
49 //!
50 //! (manual changes to the OrderedTuples.csv file)
51 //!
52 //! sort.exe OrderedTuples.csv | gawk.exe -f csv-to-json.awk > customer-name.json
53 //!
54 //! If the user wants to avoid showing hosts in that output (but still produce HBAs):
55 //!
56 //! plink.exe -l username -pw pAssw0rd 192.168.1.44 'show device-alias database' > SW44.dad
57 //! plink.exe -l username -pw pAssw0rd 192.168.1.42 'zoneshow' > SW42.zoneshow
58 //! java.exe -jar vw4tool.jar -N SW44.dad -N SW42.zoneshow -oOrderedTuples.csv
59 //! sort.exe OrderedTuples.csv | gawk.exe -v SKIPHOST=gary.yuen -f csv-to-json.awk > customer-name.json
60 //!
61 //! It is assumed the user wants to make changes to OrderedTuples.csv before the final step; otherwise, the last two steps may be combined (which avoids this script):
62 //!
63 //! plink.exe -l username -pw pAssw0rd 192.168.1.44 'show device-alias database' > SW44.dad
64 //! plink.exe -l username -pw pAssw0rd 192.168.1.42 'zoneshow' > SW42.zoneshow
